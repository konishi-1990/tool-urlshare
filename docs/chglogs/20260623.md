### 6. バグ修正

#### Bug Fix 1: テスト実行後に管理者ユーザーが消える

**原因**
`docker-compose.yml` の PHP サービスに `env_file: - ./backend/.env` が設定されていたため、`DB_CONNECTION=pgsql` が OS 環境変数として注入された。PHPUnit の `<env>` は `force="true"` がなければ既存 OS 環境変数を上書きできず、テストが実際の PostgreSQL に接続。`RefreshDatabase` が `migrate:fresh` を実行し本番 DB のデータを全消去していた。

**調査した内容**
- phpdotenv の `createMutable()` は `$_ENV` / `$_SERVER` を操作するが `getenv()` は変更しない
- OS 環境変数は `putenv()` を使わない限り PHPUnit `<env>` では上書き不可
- SQLite インメモリ判定は `DB_DATABASE === ':memory:'` で分岐しており、pgsql が注入されると `migrate:fresh` パスに入る

**修正内容**
1. `docker-compose.yml` から PHP サービスの `env_file: - ./backend/.env` を削除
2. `backend/.env.testing` を新規作成（`DB_CONNECTION=sqlite`, `DB_DATABASE=:memory:`）
3. `backend/phpunit.xml` の DB 関連 `<env>` に `force="true"` を追加
4. マイグレーションから SQLite 非互換の `DEFAULT gen_random_uuid()` を削除（HasUuids トレイトがモデルレベルで UUID を生成するため DB デフォルト不要）
5. `url_entries` の `CHECK` 制約を `if (DB::getDriverName() === 'pgsql')` で pgsql 限定に変更

#### Bug Fix 2: OGP タイトルの HTML エンティティがそのまま表示される

**症状**
URL 追加後のカードタイトルに `&#x6797;&#x4E0B;&#x8A69;&#x7F8E;` がそのまま表示される。

**原因**
OGP ページが HTML エンティティ形式（`&#x6797;` = 林）でタイトルを返す。`OgpFetchService` がそのまま DB に保存し、フロントエンドの `escHtml()` が `&` を `&amp;` に変換するため `&amp;#x6797;` となり、ブラウザが二重エスケープとして表示する。

**修正内容**
`OgpFetchService` に `decode()` メソッドを追加し、タイトル・説明の抽出時に `html_entity_decode()` を適用して DB 保存前にデコード済みテキストを格納するようにした。

```php
private function decode(string $value): string
{
    return html_entity_decode(trim($value), ENT_QUOTES | ENT_HTML5, 'UTF-8');
}
```

## 技術的なポイント・知見

- **Docker `env_file` と PHPUnit の env var 競合**: OS 環境変数は `force="true"` なしの PHPUnit `<env>` では上書きされない。`env_file` を PHP サービスに設定すると本番 DB がテストに使われる危険がある。
- **HasUuids トレイト**: UUID 生成は PHP モデルレベルで行われるため、DB の `DEFAULT` は不要。マイグレーションから削除することで SQLite 互換性が確保できる。
- **HTML エンティティの二重エスケープ**: OGP データは取得時にサーバーサイドでデコードして保存するのが正解。フロントエンドで都度処理するより DB に clean なデータを持つ方が安全。
- **Sanctum API トークン**: Laravel Sanctum は JWT ではなくランダムトークンを DB 管理。フロントは `localStorage` / Chrome 拡張は `chrome.storage.local` で保持。
