# URL共有ツール 要件定義書

## 1. プロジェクト概要

### 1.1 背景・目的

スマートフォンやPC で見つけた「あとで読みたい記事」のURLを、その場でブックマークする手間や、デバイスをまたいで参照できない問題を解決する。URLをひとつの共有リストに一元管理し、複数端末からアクセス・整理できる環境を提供する。

### 1.2 対象ユーザ

複数のデバイスを行き来しながら記事をリサーチするエンジニア・ナレッジワーカー。

### 1.3 解決する課題

| 課題 | 解決方法 |
|---|---|
| 記事を見つけてもそのまま忘れる | ワンアクションで保存できるブラウザ拡張機能 |
| 保存したURLが別端末から見られない | クラウド上の共有リストにどの端末からもアクセス可能 |
| 保存しっぱなしでリストが溢れる | 読んだ後にブックマーク化 or 削除で整理できるワークフロー |

---

## 2. 機能要件

### 2.1 URLステータス管理

URLは以下の3ステータスを持ち、遷移する。

```
仮保存 → ブックマーク
仮保存 → 削除
ブックマーク → 削除
```

| ステータス | 説明 |
|---|---|
| 仮保存 | 保存時のデフォルト。未読・評価前のURL |
| ブックマーク | 読んで保存価値ありと判断したURL |
| 削除 | 不要と判断したURL（論理削除） |

### 2.2 URL保存

| # | 要件 | 優先度 |
|---|---|---|
| F-01 | URLを入力・送信して保存できる | Must |
| F-02 | 保存時にOGP / metaタグからタイトル・概要・サムネイル画像を自動取得する | Must |
| F-03 | 保存時にステータスを選択できる（仮保存 or ブックマーク） | Must |
| F-04 | 重複URLを検出し警告する | Should |

### 2.3 共有URLリスト

| # | 要件 | 優先度 |
|---|---|---|
| F-05 | 保存されたURLをタイトル・概要・サムネイル付きで一覧表示する | Must |
| F-06 | ステータスでフィルタリングできる（仮保存 / ブックマーク） | Must |
| F-07 | 一覧からURLのステータスを変更できる（ブックマーク化 / 削除） | Must |
| F-08 | 複数端末から同じリストをリアルタイムに参照できる | Must |

### 2.4 管理画面（Web）

| # | 要件 | 優先度 |
|---|---|---|
| F-09 | 保存URLリストを表示・管理できる | Must |
| F-10 | URLを削除できる | Must |
| F-11 | ブックマーク済みURLをブラウザ用HTMLファイルとしてエクスポートできる | Should |
| F-12 | 管理者によるユーザー作成・削除・ログイン・ログアウトができる | Must |

### 2.5 ブラウザ拡張機能

| # | 要件 | 優先度 |
|---|---|---|
| F-13 | 現在のタブのURLをワンクリックで保存できる | Must |
| F-14 | 保存時にステータスを選択できる | Must |
| F-15 | 拡張機能内で保存済みリストを確認できる | Could |

### 2.6 API

| # | 要件 | 優先度 |
|---|---|---|
| F-16 | URL登録エンドポイント（POST） | Must |
| F-17 | URLリスト取得エンドポイント（GET） | Must |
| F-18 | URLステータス更新エンドポイント（PATCH） | Must |
| F-19 | URL削除エンドポイント（DELETE） | Must |
| F-20 | Sanctum API トークンによる認証 | Must |
| F-21 | 管理者によるユーザー管理（一覧・作成・削除） | Must |

---

## 3. 非機能要件

| 分類 | 要件 |
|---|---|
| マルチデバイス | スマートフォン・PCのブラウザで動作すること |
| レスポンシブ | モバイルサイズにも対応したレイアウト |
| セキュリティ | 認証済みユーザのみ自分のリストにアクセスできること |
| OGP取得 | サーバサイドでURLのメタ情報を取得する（CORSを回避するため） |

---

## 4. スコープ外（v0.1）

以下はv0.1の対象外とする。

- タグ・フォルダによる分類
- ユーザ間での共有リスト（マルチユーザ共有）
- 全文検索
- モバイルアプリ（iOS / Android）
- 読了時間の推定・進捗管理

---

## 5. データモデル（概要）

### User

| フィールド | 型 | 説明 |
|---|---|---|
| id | UUID | ユーザID |
| email | string | メールアドレス（ログインID） |
| password_hash | string | パスワードハッシュ |
| is_admin | boolean | 管理者フラグ |
| created_at | datetime | 作成日時 |

### UrlEntry

| フィールド | 型 | 説明 |
|---|---|---|
| id | UUID | エントリID |
| user_id | UUID | 所有ユーザID |
| url | string | 保存したURL |
| title | string | 自動取得したタイトル |
| description | string | 自動取得した概要 |
| thumbnail_url | string | 自動取得したサムネイル画像URL |
| status | enum | `temporary` / `bookmarked` / `deleted` |
| created_at | datetime | 保存日時 |
| updated_at | datetime | 最終更新日時 |
